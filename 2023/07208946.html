如何搭建一个Hexo

这篇总结主要讲解了如何搭建一个hexo

一.初次准备

1.进入Hexo官网

2.全局安装hexo脚手架

npm install hexo-cli -g   //全局安装hexo-cli脚手架
hexo init blog            //创建一个名为blog的文件夹，这个可以任意命名
cd blog                   //切换至这个文件夹，你也可以进去编译软件，
                          //则无需进入，但必须保证blog为输入命令行时的根目录
npm install               //下载依赖包
hexo server               //运行hexo，也可以写成缩写，hexo s

这里你完成以后，你就可以看见原始的hexo的博客页面

二.关联gitee

1.完成关联，进入国内gitee中，新建一个仓库，配置deploy

deploy:
  type: git          # 类型为git      
  repo:              # 这里写你的Gitee仓库地址
  branch: master     # 这里写你的分支

2.创建git仓库 git init

3.下载一键部署插件

npm install hexo-deployer-git --save

4.检测是否有git

git status  //如果发现没有，再执行一遍git init

5.上传部署

hexo c && hexo g && hexo d
// hexo c
// hexo g
// hexo d

6.此时你会发现你的代码已经上传上去了

三.开启Pages，完成部署

1..点击管理，允许公开–>服务中的gitee pages–>选中启用https，并启动

此时他会给你一个网址

注意：以后你每次的代码上传部署，都需要重新在这里面点击更新

2.在你的Hexo中的_config.yml文件中，将你的URL的地址改为page给你的地址，并配置root选项

url: 这里写Page给你的地址
root: 这里写Page给你的地址中最后的文件路径，要写出这样  /文件路径/

什么是文件路径你们应该分的清楚吧，https协议：//主机ip:端口号/文件路径/

hexo c && hexo g && hexo d进行上传部署 –> 更新pages –> 此时访问Pages中的地址即可访问你的博客

四.美化主题

1.使用hexo-theme-matery替代原生页面，hexo-theme-matery配置文档

2.下载hexo-theme-matery包

git clone https://github.com/blinkfox/hexo-theme-matery.git

这里可能下载不好，由于你的网络等出现问题，同时不要去他的文档中下载稳定版，因为他的稳定版是缺少script等几个文件夹的。因为git在拉取或者提交项目时，中间会有git的http和https代理，但是我们本地环境本身就有SSL协议了，所以取消git的https代理即可，不行再取消http的代理

解决：

//取消http代理
git config --global --unset http.proxy
//取消https代理 
git config --global --unset https.proxy

再去执行克隆拉取代码，如果仍然不成功，由于github并非国内的，则需要科学上网。你多试几次一定能成功的。

3.当你把hexo-theme-matery包下载完毕以后，将它放在Hexo下的themes文件夹中

4.修改Hexo根目录的_config.yml 的 theme 的值：theme: hexo-theme-matery

5.根据官方文档自己去配置文件即可

hexo-theme-matery配置文档

我说几个可能产生的问题

1.取消首页渐变颜色动画
在themes\Matery\source\css\matery.css，ctrl+F快捷键查找.bg-cover:after，注释掉即可。

2.改变导航栏的绿色
打开 themes/matery/source/css/matery.css 文件，找到bg-color 这个属性，即为导航栏和底部的颜色

3.取消底部说明
直接打开页面，找到他的类名，去搜索这个类名，找到这个footer.ejs文件，给他注释或者删除

4.对于关于页面的底部可视化图表
找到post-charts.ejs,对他的可视化的HTML结构进行注释

5.代码块{}括号乱码

npm uninstall hexo-prism-plugin 来卸载掉它

修改 Hexo 根目录下 _config.yml 文件

highlight:
  enable: false
  line_number: true
  auto_detect: false
  tab_replace: ''
  wrap: true
  hljs: false
prismjs:
  enable: true
  preprocess: true
  line_number: true
  tab_replace: ''

6.你如果没下载hexo-prism-plugin，可能会出现代码大小不一致，对不齐

在你的代码中，找到你不一致的文档，对他的代码块**``````**,将他的语言在重新写一遍，没有定义语言的，则把这几个点重新写一遍。

五.上传至Github

之前配置的不要

这里都给他注释掉
url: 这里写Page给你的地址
root: 这里写Page给你的地址中最后的文件路径，要写出这样  /文件路径/

将你的deploy修改

deploy:
  type: git          # 类型为git      
  repo:              # 这里写你的Github仓库地址
  branch: master     # 这里写你的分支

可能会出现上传不上去的问题
1.换检查网络
2.使用SSH网址进行上传
